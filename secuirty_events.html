Security events

Help

This is a requests view; you can use normal Data Explorer query syntax. These query fragments are generally useful in all requests views:

  • ~open and ~closed: open and closed requests, respectively
  • ~requested-by-me: requests for which you are the requester
  • ~assigned-to-me: requests on which your action is needed
  • id=150: a request with a specific ID
  • type=laptop: laptop orders (if available in this view)
  • state.display:Delivered: requests with "Delivered" in their state display text
  • created~"^2022-03": requests created in March 2022
  • updated>=2022: requests updated in 2022 or later

This view shows security events (opened manually or via admin hooks).
